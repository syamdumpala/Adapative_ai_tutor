# Status Docs

This folder tracks the **current state** of the frontend: what is built, what
works, what is in progress, and what the known gaps are.

- [`STATUS.md`](./STATUS.md) — the living status snapshot.

## How to use

Update [`STATUS.md`](./STATUS.md) **after every change** so it always reflects
reality. It answers, at a glance:

- What features/modules exist and are they working?
- What tooling and guardrails are in place?
- What is in progress or blocked?
- What are the known gaps / TODOs?

`STATUS.md` is a *snapshot* (present tense, overwrite as things change). The
chronological log of how we got here lives in
[`../history/HISTORY.md`](../history/HISTORY.md).
