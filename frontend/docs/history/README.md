# History Docs

An **append-only** changelog of everything done in the frontend. Newest entries
go at the top. Never rewrite or delete past entries — they are the historical
record.

- [`HISTORY.md`](./HISTORY.md) — the log.

## How to use

Append a new entry for **every** change (this is mandatory — see
[`../../RULES.md`](../../RULES.md) §2). Each entry should capture:

- **Date** (`YYYY-MM-DD`).
- **What** changed (concise).
- **Why** (motivation / decision), when not obvious.
- **Links** to the files or other docs affected.

`HISTORY.md` is the chronological record; the present-tense snapshot of "where
we are now" lives in [`../status/STATUS.md`](../status/STATUS.md).
